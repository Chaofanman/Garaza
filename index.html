First Week

In the first week, we sent our mockups to our client for review and did a dummy user sign up and login. Since it was a dummy sign up and login, we first used sqlite3 as a database. We used an MD5 hashing function for the passwords. We also deployed this on Heroku for testing and as a foundation for the future.

Second Week

In the second week, our client finalized our mockups. A lot of time was spent modularizing the database, tables, and their associations. We also rewrote our sign up via email and login to match the individual consultant fields; passwords were still hashed using an MD5 hashing function. We also moved to a postgres database for future proofing.

Third Week

For the third week we focused on finishing all front end elements. This is the biggest part of the first module and we plan on finishing this the next few weeks. Most of our groups efforts would be directed here for now.

Fourth Week

We have made good progress in the the front end pages. Some of them are completed while others have the barebones markup. We've had trouble sending the different kinds of requests to the server since OpenUI5 handles data binding differently. This week we will focus on the requests to the back end server and connecting it to the front end fields to make the page more dynamic. Refactoring of the back end is needed since the page to edit and sign up will be one big page so the data should be sent as a single request as opposed to separate ones.

After some talks with out client, we both agreed that switching our views from javascript to xml would be the best move going forward. Most exampled on the sapui5.explored website were in xml and it was difficult having to translate that code into js. Since we are now using xml as well it would be easier for us to follow the examples since translation is no longer required. However we had to redo all of our current views which would set us back a little.

Fifth Week

We're now about halfway done with all of the front end views we need to do. We have started to work on connecting the database with our frontend since we have currently just been using locally stored data. Along with this, we will start working on user authentication, giving different accounts different restrictions on what they can or cannot do.

Sixth Week

This week we were able to successfullly send RESTful requests to the live Heroku server in the SAP client code. The consultant page markup and design is done and the EOI page is almost as well. We are working on making the page's content to be more dynamic through the RESTful requests. In the backend, we have started the filtering and querying the users through sectors, expertise, and employment. We also need to change the edit page as one big request as opposed to separate requests by field. In the frontend we are also doing the vendor page and the login page.

Test Cases

Currently we have been testing the login system back end by inputting invalid and valid inputs and checking if they would output the desired results. We will also begin to test the sql injection counter-measures we have in place.

We have been testing page navigation as well making sure we are being taken to the right pages.
